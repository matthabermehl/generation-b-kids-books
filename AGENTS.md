# Agent Operating Map (Codex)

This repository is configured for long-running autonomous execution. The repository is the system of record for state, constraints, and evidence.

## Golden Rules
- Work one task at a time from `.agent/feature_list.json`.
- Never set `passes=true` without evidence in the same session.
- Use `scripts/agent/` commands instead of ad-hoc command variants.
- Leave the repository clean at session end: quality gates run, progress log appended, commit created.

## Start-of-Session Protocol
1. Read `.agent/progress.log` (latest block first).
2. Inspect `.agent/feature_list.json` for failing high-priority tasks.
3. Run `bash scripts/agent/smoke.sh`.
4. If smoke fails, run `debug-triage` before feature work.
5. Write/update `.agent/current_task.md` for exactly one task.

## Skill Routing
- Session orientation: `session-startup`
- Single-task implementation: `implement-one-feature`
- Baseline/test failures: `debug-triage`
- User-facing flow validation: `e2e-validate`
- PR packaging: `pr-driver`
- Documentation drift: `doc-gardener`
- Entropy cleanup: `garbage-collector`

## Script Interfaces
- `bash scripts/agent/init.sh`
- `bash scripts/agent/smoke.sh`
- `bash scripts/agent/test.sh`
- `bash scripts/agent/lint.sh`
- `bash scripts/agent/format.sh`
- `bash scripts/agent/quality.sh`

- `bash scripts/agent/e2e.sh`
- `bash scripts/agent/snapshot_ui.sh <url>`
- `scripts/agent/feature_list.py list`

## Source of Truth
- Product: `docs/PRODUCT.md`
- Architecture: `docs/architecture.md`
- Security posture: `docs/SECURITY.md`
- Reliability/escalation: `docs/RELIABILITY.md`
- Quality rubric and trend: `docs/QUALITY_SCORE.md`
- Active execution plans: `docs/exec-plans/active/`
- Completed execution plans: `docs/exec-plans/completed/`
- Design principles: `docs/design-docs/core-beliefs.md`
- External/vendor references: `docs/references/`

## Escalation Requirements
Ask for human input when:
- Requirements are conflicting or ambiguous after reading the source docs.
- The change crosses auth, payment, destructive data operations, secrets, or outbound networking policy boundaries.
- Repeated attempts fail without new technical evidence.
